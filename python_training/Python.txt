Variables:
- Memory representation to store or access a value
- No default values for variable (will throw name error)
- Creates memory space --> Stores value --> Variable name is assigned

Datatypes:
- the type of data which is stored; helps us to decide what operations can be done with this data and how much memory is allocated for it
--> Integer (int; no fixed size) - small int (28 bytes), large int (dynamically assigned)
--> Float (24 bytes; +-1.8*10^308) - precision (15 to 17 digits can be stored after point)
--> Complex - stores imaginary values (32 bytes) Eg:5+6j where j is an imaginary number.
--> String (str; size depends on chars stored)
--> Boolean (bool; True(1)/False(0); 28 bytes)
--> Sequence - like Arrays to store multiple values. (List->mutable;[];56 bytes, Tuple->immutable;();40 bytes)
--> Set - unordered, unique collection; {}
--> Dictionary - key value pairs; 240 bytes initally

=== Day 6 ============================= 
py unit - Python Unit Testing

- SRS: Software Requirement Specification - Document should be created before starting the development of the project which contains all the requirements and specifications of the project

- Test Cases: should be created based on the requirements mentioned in the SRS document to test

- STLC: Software Testing Life Cycle - process of testing the software which includes different phases
--> Phases of STLC: Requirement Analysis, Test Planning, Test Case Development, Environment Setup, Test Execution, Bug Reporting, Test Closure

- Types of Testing
--> Functional Testing - Testing based on SRS requirements
|---> Unit Testing - Individual functions tested separately
|---> Integration Testing - Combined modules tested
|---> System Testing - Complete application tested
|---> Acceptance Testing - Client tests before the application is launched
--> Non Functional Testing - Testing for application support but not in SRS requirements 
|---> Performance - Testing the application performance based on speed and stability

- Testing Techniques
--> Black Box - Tester will only be able to see the output but not the internal code
--> White Box - Tester will check for the internal code logic
--> Grey Testing - Tester will know the internal code partially 

- Manual (best for UI-based applications) & Automation Testing (best for scripts)

- Autonomous Testing: AI Tests automatically without human intervention.
